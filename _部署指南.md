# 部署到 GitHub Pages · 10 分钟流程

## 前置准备
- GitHub 账号（没有就去 github.com 注册）
- 安装 git（Mac：`brew install git` 或访问 git-scm.com）

## 步骤

### 1. 在 GitHub 新建仓库
- 登录 github.com → 右上角 `+` → New repository
- Repository name: `beiya-p2`（或你想要的名字）
- 勾选 `Public`
- **不要** 勾选"Add a README file"
- 点 Create repository

### 2. 推送本文件夹到 GitHub
在本文件夹路径下打开 Terminal，运行（把 YOUR_USERNAME 换成你的）：

```bash
cd "~/Documents/Claude/Projects/综合性 个人口语网站或者应用/_github_pages_deploy"
git init
git add .
git commit -m "北雅 P2 口语素材宇宙 v1"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/beiya-p2.git
git push -u origin main
```

首次 push 会让你登录 GitHub · 按提示做。

### 3. 开启 GitHub Pages
- 回到仓库页面 → Settings → Pages
- Source: 选 `Deploy from a branch`
- Branch: 选 `main` · 文件夹 `/ (root)`
- Save

等 1-2 分钟后，你的网站就在：
`https://YOUR_USERNAME.github.io/beiya-p2/`

### 4. 分享
复制这个 URL 发给朋友。微信打开（手机可能有点挤，桌面端最佳）。

## 更新网站
以后你改了内容，想更新：

```bash
cd "~/Documents/Claude/Projects/综合性 个人口语网站或者应用/_github_pages_deploy"
git add .
git commit -m "更新描述"
git push
```

1-2 分钟后网站自动更新。

## 常见问题

**Q: 国内访问 github.io 慢吗？**
A: 偶尔慢，但可用。想更快可以后续换国内 CDN（Netlify / Vercel / 阿里云 OSS）。

**Q: 会被 google 搜到吗？**
A: GitHub Pages 是公开的，任何人都能访问。如果你不想被搜到，可以在每个页面加 `<meta name="robots" content="noindex">` 阻止搜索引擎。

**Q: 学生的收藏/进度数据会共享吗？**
A: 不会。当前版本所有数据都存在**每个学生自己的浏览器 localStorage**，相互隔离。学生换浏览器/清缓存数据会丢。

**Q: 你（老师）在编辑器里改的内容学生能看到吗？**
A: 不能。编辑器也是存本地 localStorage——你改的内容只在你这台电脑看得到。如果要学生看到新版本的内容，你需要：
  1. 在编辑器里点"导出全部修改 JSON"
  2. 把 JSON 发给 Claude，我帮你合入正式的 tiers.js
  3. `git push` 更新网站 → 学生刷新就看到了
